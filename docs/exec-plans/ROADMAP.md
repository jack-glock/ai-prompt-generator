## 작업 흐름

1. ROADMAP에서 항목 선택 → 작업 시작 (TodoList 생성).
2. 코드·문서 동시 갱신:
   - 코드 → `app/page.tsx` 또는 `lib/promptBuilder.ts`
   - 문서 → `FEATURES.md` 상태 변경 + 필요 시 design-docs 갱신
3. 완료 → 이 파일에서 항목 제거 + `git push` (Vercel 자동 배포).
4. 보류 결정 → `BACKLOG.md`로 이동하면서 사유 기록.
5. 큰 변경은 DevSync 측 `CHANGELOG.md` 에도 1~3줄 추가 검토.
