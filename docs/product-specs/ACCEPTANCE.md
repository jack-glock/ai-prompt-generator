# ACCEPTANCE — 수용 기준

> "기능을 만들었다"가 아니라 "**작동한다고 인정할 조건**". 새 릴리즈 전에 이 표를 통과해야 한다.

## 1. 프롬프트 생성 (핵심 기능)

- [ ] 비어있는 한글 요청 + 기본 옵션 → 5개 결과 박스 모두 비어있지 않은 영어 출력
- [ ] 한글 요청 100자 입력 → GPT/Nano Banana/Midjourney 결과 모두에 한글이 따옴표로 포함됨
- [ ] 비율 `1000x600` 선택 → Midjourney 결과 끝에 `--ar 5:3`
- [ ] 비율 `직접 입력` + `1920x1080` → Midjourney 결과 끝에 `--ar 16:9`
- [ ] 비율 `직접 입력` + `21:9` → Midjourney 결과 끝에 `--ar 21:9`
- [ ] 금지 요소 체크 변경 → 1초 이내 결과에 반영
- [ ] 참고 이미지 업로드 → GPT/Nano Banana 프롬프트에 reference 언급 포함
- [ ] 참고 이미지 역할 변경 → 즉시 결과 반영

## 2. UI

- [ ] 데스크톱 1280px 이상 → 좌우 2열로 표시
- [ ] 데스크톱 1024px 이하 → 1열로 자동 전환
- [ ] 칩 클릭 → 즉시 색 반전 (지연 없음)
- [ ] 복사 버튼 클릭 → 1.5초간 "복사됨" 표시 후 원상복귀
- [ ] 초기화 버튼 → 모든 입력 + 업로드 이미지 초기화
- [ ] 저장 버튼 → `prompt-*.txt` 파일 다운로드 시작

## 3. 안정성

- [ ] `npm run build` 무경고 통과
- [ ] Vercel 빌드 90초 이내 성공
- [ ] 라이브 사이트 200 OK 응답
- [ ] 한글 입력 깨짐 없음
- [ ] 새로고침 후 첫 화면 로드 3초 이내

## 4. 보안

- [ ] 업로드 이미지가 네트워크 탭에 외부 요청으로 잡히지 않음
- [ ] 콘솔에 경고/에러 로그 없음
- [ ] `.env*` 파일이 GitHub에 노출되지 않음
- [ ] `package.json` 변경 시 `npm audit` 결과에 critical 없음

## 5. 문서

- [ ] 새 기능 추가 시 `FEATURES.md` 즉시 갱신
- [ ] 의도적 미구현은 `BACKLOG.md`에 사유 명시
- [ ] 날짜 표기 YY-MM-DD 일관성
- [ ] DevSync 측 CHANGELOG에 큰 변경 한 줄 추가 검토
