# 지식 맵 구조 (AXIS 1)

> 26-04-29 정리. 저장소 문서를 어떻게 운영하는지에 대한 원칙 가이드.

## 한 줄 요약

**`AGENTS.md`는 모든 지식을 담는 문서가 아니라, 필요한 위치로 안내하는 거점**이다. 상세한 내용은 `docs/` 하위 폴더에 누적하고, 에이전트는 필요한 문서만 점진적으로 펼쳐 읽는다.

## 저장소 구조

```
REPO ROOT
├─ AGENTS.md              짧은 ToC / 맵 (진입점)
├─ ARCHITECTURE.md        최상위 구조 설명
├─ docs/                  상세 지식 저장소
│  ├─ design-docs/        설계 문서
│  ├─ product-specs/      수용 기준
│  ├─ model-specs/        모델 사양 (이 프로젝트 추가 폴더)
│  ├─ exec-plans/         실행 계획
│  └─ references/         외부 레퍼런스
├─ QUALITY_SCORE.md       품질 루브릭
└─ SECURITY.md            보안 규칙
```

## 세 가지 운영 원칙

### 1. ROOT — 루트는 안내판

`AGENTS.md`는 모든 상세를 담는 문서가 아니다. **어디를 먼저 읽고 무엇을 피해야 하는지 알려주는 진입점**이다. 따라서 `AGENTS.md`는 짧고 간결해야 하며, 세부 내용은 `docs/` 하위로 보낸다.

### 2. RECORD — 기록은 하위 폴더에

실제 기준 문서는 `docs/` 아래에 분리해 둔다. 설계 문서, 수용 기준, 실행 계획, 참고 자료가 이곳에 누적된다.

| 폴더 | 담는 내용 |
|---|---|
| `design-docs/` | 설계 문서 (UI 디자인, 프롬프트 전략 등) |
| `product-specs/` | 기능 명세, 수용 기준 |
| `model-specs/` | 모델별 사양 (GPT Image / Nano Banana / Midjourney / Niji) |
| `exec-plans/` | 로드맵, 백로그, 실행 계획 |
| `references/` | 외부 자료, 참고 문헌 |

### 3. DISCLOSURE — 필요할 때만 펼쳐 읽기

에이전트는 처음부터 모든 문서를 읽지 않는다. **필요한 하위 문서만 점진적으로 열어 컨텍스트 예산을 아낀다**. 이를 위해 `AGENTS.md`(또는 `CLAUDE.md`)에는 어느 폴더에 무엇이 있는지 짧은 ToC만 두고, 깊은 내용은 그 폴더 안 문서에서만 보도록 유도한다.

## 이 프로젝트(AI Prompt Generator)에 대한 적용

현재 진입점은 두 가지로 운영 중:

- **`AGENTS.md`** — 외부 협업자(다른 에이전트)용 진입점
- **`CLAUDE.md`** — 다음 Claude 세션 인덱스 (현재 v0.5/v0.6 라이브 상태 추적)

두 파일 모두 짧게 유지하고, 자세한 내용은 `docs/` 하위로 분산되어 있어야 합니다. 각 폴더의 현재 내용은 `CLAUDE.md`의 "docs 지도" 섹션 참조.

## 실무 체크리스트

새 문서를 만들 때:

1. **루트(`AGENTS.md` / `CLAUDE.md`)에는 짧은 링크만** — 본문은 추가하지 않는다
2. **상세 본문은 `docs/` 하위 적절한 폴더로** — 카테고리(design / product / exec / refs / model)에 맞춰 분류
3. **읽는 사람에게 지도를 먼저 보여주고, 본문은 따로 열도록** — 한 문서에 너무 많이 담지 않기
